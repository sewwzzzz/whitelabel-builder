# Naive UI 国际化

# bug

- 描述：在切换语言后，naive ui 语言组件的 button 样式没有变化，这是正常的，因为 getButtonLocale 并没有依赖响应式；在切换主题时，naive ui 区域组件和语言组件的 button 样式都更新了

- 解决：问题出自 naive ui 组件 n-config-provider 的主题切换，由于响应式数据变化后，内部的 naive ui 组件接收到变化的主题后会重新渲染，所以所有 button 样式都能够更新；如果是语言切换，NButton 并没有接收该响应式数据，所以也不会重新渲染，只有接收该响应式数据的 Naive 组件才会重新渲染

- onUpdated 钩子只在有实际 DOM 变化时才会触发，重新渲染时不一定触发
