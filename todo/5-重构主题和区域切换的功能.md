# 职责划分不清

- App.vue 中在调用 manager 的方法上封装了相关的操作型 provide，但是 App.vue 只需要变量展示，当前职责太多

# 渲染展示问题

- ThemeChange 的组件 n-button 的 type 属性中未与响应式变量关联，如果没有全局渲染，无法及时更新按钮的 type 属性对应的样式

# 结论

- 使用 pinia 重构，将职责集中于 pinia 中，pinia 中维护的变量是响应式的，且全局可访问，解决了当前问题

# pinia

- 承载全局状态

- state、getter、action (ref()、computed()、function())

- 启用标准：可以在整个应用中访问的数据
